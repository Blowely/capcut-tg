# Руководство по внесению вклада

Спасибо за интерес к проекту CapCut Telegram Mini App! Мы рады любому вкладу.

## Как внести вклад

### 1. Подготовка окружения

```bash
# Форкните репозиторий и клонируйте
git clone https://github.com/yourusername/capcut-tg.git
cd capcut-tg

# Установите зависимости
npm install
cd apps/backend && npm install && cd ../..
cd apps/frontend && npm install && cd ../..

# Запустите локально
npm run dev
```

### 2. Создание ветки

```bash
git checkout -b feature/your-feature-name
# или
git checkout -b fix/bug-description
```

### 3. Разработка

- Следуйте существующему стилю кода
- Пишите понятные commit сообщения
- Добавляйте комментарии к сложному коду
- Обновляйте документацию при необходимости

### 4. Тестирование

Убедитесь, что ваш код работает:

```bash
# Проверка линтера
npm run lint

# Сборка
npm run build
```

### 5. Создание Pull Request

1. Push в ваш форк
2. Создайте Pull Request в основной репозиторий
3. Опишите изменения в PR описании
4. Дождитесь ревью

## Стиль кода

### TypeScript

- Используйте строгую типизацию
- Избегайте `any` где возможно
- Используйте интерфейсы для объектов

### React

- Используйте функциональные компоненты
- Используйте хуки вместо классов
- Следуйте принципам композиции

### Именование

- **Файлы**: PascalCase для компонентов, camelCase для утилит
- **Переменные**: camelCase
- **Константы**: UPPER_SNAKE_CASE
- **Интерфейсы**: PascalCase без префикса I

### Коммиты

Используйте conventional commits:

```
feat: добавлена функция обрезки видео
fix: исправлена ошибка загрузки файлов
docs: обновлена документация API
style: исправлено форматирование кода
refactor: рефакторинг модуля проектов
test: добавлены тесты для AuthService
chore: обновлены зависимости
```

## Отчеты об ошибках

При создании issue включите:

1. Описание проблемы
2. Шаги для воспроизведения
3. Ожидаемое поведение
4. Фактическое поведение
5. Версии (Node.js, npm, браузер)
6. Логи ошибок

## Предложения функций

При предложении новых функций:

1. Опишите проблему, которую решает функция
2. Предложите решение
3. Рассмотрите альтернативы
4. Добавьте примеры использования

## Код поведения

- Будьте уважительны к другим участникам
- Конструктивная критика приветствуется
- Помогайте новичкам
- Фокусируйтесь на коде, а не на людях

## Лицензия

Внося вклад, вы соглашаетесь, что ваш код будет лицензирован под MIT License.
